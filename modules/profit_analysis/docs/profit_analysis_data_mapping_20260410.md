# Profit Analysis Data Mapping

- Date: `2026-04-10`
- Scope: map profit-analysis V1 required fields to current repository data assets
- Status: `ready_for_implementation_planning`

## Bottom Line

The current repository already has enough data to start a `prototype-level` profit analysis module.

Fields that are already in usable shape:

- current inventory
- inventory snapshot date
- product identity
- style/category metadata
- price proxy
- first-order quantity
- 30-day prediction output can be produced by the current tree model

Fields that are only partially available or need a derived proxy:

- inbound within 30 days
- lifecycle end date
- lead time

Fields that are currently missing for a production-grade version:

- unit cost
- holding cost
- salvage value
- stockout penalty

## Fields That Need Business Confirmation

### Lifecycle

Candidate fields exist:

- `PL_CYCLE`
- `LISTING_DATE`

But the following must be clarified before formal use:

- does `PL_CYCLE` mean style lifecycle length or planned selling cycle
- what unit is it in
- can it be trusted as a current operational lifecycle field

Until that is confirmed:

- do not make lifecycle a hard constraint in V1
- allow it as an optional soft cap only

### Inbound

The current project has supply-related signals, but not a clean final business field for:

- `confirmed in-transit quantity arriving within the next 30 days`

So before formal use, the team should confirm:

- whether `qty_inbound` can serve as a stable proxy
- whether a better operational inbound table exists
