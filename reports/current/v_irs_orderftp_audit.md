# V_IRS_ORDERFTP 原始订单表审计报告

源文件：
- `data_warehouse/fact_orders/V_IRS_ORDERFTP.csv`

相关清洗入口：
- `src/etl/clean_data.py`

## 1. 审计结论

`V_IRS_ORDERFTP.csv` 可以作为当前项目的 canonical raw order table 之一，但它不是可以不加业务定义就直接当成最终训练标签的“干净事实表”。

这张表的主要问题有 3 类：

1. `TYPE` 存在系统性缺失，不是偶发空值。
2. `QTY` 存在负数，且大概率不是“负需求”，而是冲销/调整类事件。
3. 存在完全重复行，说明它更像业务导出的事件流水，而不是 fully reconciled 的净事实表。

当前 ETL 已经在使用这张表，但使用方式比较保守：
- 新表清洗逻辑会把 `QTY < 0` 直接截到 `0`
- 再按 `TYPE == 现货 / 期货` 拆成 `qty_replenish / qty_future`
- `TYPE` 缺失行在当前逻辑下基本不会进入最终训练标签

这意味着：
- 当前训练链路可以继续跑
- 但 `TYPE` 缺失和负数 `QTY` 的业务语义，仍然值得在后续优化前优先澄清

## 2. 字段语义

表结构只有 6 列：

| 字段 | 推定语义 |
| --- | --- |
| `NAME` | `style_id` / 款号 |
| `NO` | `sku_id` / 条码级 SKU |
| `STORENAME` | 买手/门店编码 + 门店名，格式类似 `RA0136-浙江嘉兴李燕` |
| `BILLDATE` | 业务日期，格式 `YYYYMMDD` |
| `QTY` | 订单数量 |
| `TYPE` | 订单类型，当前主要是 `现货 / 期货` |

这个语义判断有数据支持：

- `NAME` 去重：`3754`
- `NO` 去重：`12084`
- 一个 `NAME` 平均对应 `3.22` 个 `NO`
- 一个 `NO` 只对应 `1` 个 `NAME`

因此：
- `NAME` 更像 style 级编码
- `NO` 更像颜色尺码级 SKU 编码

## 6. 负数 `QTY` 的问题

| 指标 | 数值 |
| --- | ---: |
| 负数行数 | `995` |
| 负数数量合计 | `-1,562` |
| 属于 `现货` 的负数 | `994` |
| 属于 `TYPE` 缺失`的负数 | `1` |

样本显示，这些负数行很像：
- 冲销
- 撤单
- 改单
- 调整

而不像“真实负需求”。

因此当前更合理的业务判断是：

- `QTY < 0` 不应直接视为真实补货需求
- 应单独视为 `negative_adjustment` 或类似事件

## 8. `STORENAME` 的含义

`STORENAME` 统一格式为：

- `RA0529-河北唐山孙亚军`
- `RA0250-江苏靖江姚美霞`
- `RA0938-北京朝阳MISS AS`

结论：
- `RAxxxx` 是买手/门店统一编码
- 后面是区域 + 门店/买手名称

可直接作为当前项目里的稳定 `buyer_id` 使用。

活跃度较高的店包括：

- `RA0529-河北唐山孙亚军`
- `RA0250-江苏靖江姚美霞`
- `RA0097-江苏苏州仲亚云`
- `RA0004-浙江台州崔林燕`
- `RA0938-北京朝阳MISS AS`
- `RA0721-四川自贡徐黄鹂`

## 9. 当前 ETL 如何使用这张表

当前项目在 `clean_orders_new()` 中直接读取这张表。

现有关键处理逻辑是：

1. `QTY` 转数值
2. `QTY < 0` 直接 `clip(lower=0)`
3. `TYPE == 现货` 记入 `qty_replenish`
4. `TYPE == 期货` 记入 `qty_future`
5. 聚合到 `(buyer_id, sku_id, order_date)`

这是一种保守处理，优点是稳，缺点是：
- `TYPE` 缺失信息被丢失
- 负数事件的业务语义没有被保留下来

## 10. 对项目的直接影响

### 当前可以继续使用的部分

- `QTY > 0`
- 且 `TYPE` 明确为 `现货 / 期货`

这部分完全可以继续作为正式训练来源。

### 当前必须澄清的部分

1. `TYPE` 缺失到底代表什么
2. `QTY < 0` 到底代表什么业务动作
3. 重复行到底是重复导出，还是多次真实业务动作

如果不澄清：
- 会损失一部分有效订单信息
- 也会影响后续 `phase8` 的弱信号 rescue 和 zero-true suppression
